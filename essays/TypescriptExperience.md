## Typescript so far
I think that Typescript wasn't that hard to learn so far because I can use a lot of javascript in typescript. One thing that I liked about typescript that javascript didn't have was the automatic error checking. Sometimes it got in the way because it would show the error before I was done writing and it would confuse me at the time but I think that it is useful because you don't have to run your code to check if there are errors. 

## Typescript expectations
I didn't have any prior experience using typescript but learning the basics of javascript in the earlier assignment was also useful for learning typescript because you can just use javascript in typescript. I think that typescript is useful from a software engineering perspective because you can use your knowledge from javascript with some of the features that typescript adds on similar to how you can use c in c++ but have access to classes and objects.

## Athletic Software Engineering 
I believe that the Athletic Software Engineering appraoch is useful to gaining the skills that are expected. Instead of having on homework assignment a week and spending a lot of time creating a whole project, learning the individual skills one by one can be more useful. I think that this is a useful approach because you can learn all of the skill needed to complete a project instead of learning to learning the skills at the same time you will need to implement them when creating a project. So far, I was able to complete the WOD because they were fairly simple and didn't take that long to come up with the solution after knowing how the problem should be solved.

## WOD
In the description of the WOD it says that skills in being correct and being efficient would be improved. I believe that these are the two most important skills in software engineering because if your product cannot function then it doesn't serve its only purpose and if you aren't efficient then it doesn't matter if your product or project works well if you can't get it done within a time limit. I believe that once a week homework assignments or creating entire programs don't help improve being efficient. You may have a deadline of a week to get it done but there isn't a timed portion where you have to complete part of the program and ensure that it works. But if the tasks are small you can improve your efficiency by competing it within the time limit and ensure that it's correct becuase you can test it right away.
